# NBR Audit Checker (tin-findr)

Live: **https://check-tin.asif.dev** · How it works: **https://check-tin.asif.dev/how-it-works**

A privacy-first, 100% client-side tool to check whether a Bangladesh
Taxpayer Identification Number (TIN) appears on the National Board of
Revenue's Risk-Based Audit Selection list for Assessment Year 2023–24.

The full list (87,685 records, merged from both officially published
lists) ships as a static `audit.json` file. Lookups happen entirely in
the browser — your TIN is never sent over the network.

## Highlights

- **No backend lookup.** TINs are matched in-memory against `audit.json`.
- **Two source lists merged.** 72,196 from the 49-zone master list +
  15,489 from the 8-zone supplementary list = 87,685 total records.
- **Source flag per record.** Each entry tracks whether it came from
  list 1, list 2, or both.
- **Privacy-respecting analytics.** Self-hosted Umami at
  `agent-analytics.asif.dev` — no cookies, no third-party trackers,
  no PII. The TIN string itself is never tracked; only event counts.
- **Live counters.** A small server function (`src/server/umami.functions.ts`)
  reads aggregate visit and `tin_lookup` event counts from the Umami
  Admin API and shows them in the header strip.

## Data shape

`public/data/audit.json` is a single compact object:

```ts
type AuditDB = {
  zones:   string[];                                          // zone names (deduped)
  circles: string[];                                          // circle names (deduped)
  data: Record<
    string,                                                   // TIN → tuple
    [zoneIdx: number, circleIdx: number, submissionType: string, assessmentYear: string, source?: 0 | 1 | 2]
  >;
};
```

`source` encodes which official list the record came from:
`0` = list 1 (49-zone master), `1` = list 2 (8-zone supplementary),
`2` = both.

### Environment variables (optional)

The visit/lookup counter uses the Umami Admin API. Without these, the
site still works — it just hides the counter strip.

```
UMAMI_HOST=https://agent-analytics.asif.dev
UMAMI_WEBSITE_ID=<umami-website-id>
UMAMI_USERNAME=<umami-admin-username>
UMAMI_PASSWORD=<umami-admin-password>
```

For local dev, put these in `.dev.vars` (already gitignored). For
production, set them as Cloudflare Worker secrets.

## Deployment

This repo is wired to **Lovable** — pushes to `main` are deployed
automatically. The build target is Cloudflare Workers, configured by
`wrangler.jsonc` and the `cloudflare` plugin bundled inside
`@lovable.dev/vite-tanstack-config`.

Do **not** add `tanstackStart`, `viteReact`, `tailwindcss`,
`tsConfigPaths`, or `cloudflare` plugins manually to `vite.config.ts` —
the Lovable preset already includes them and duplicating will break
the build.
